# Implementation Execution Law

## 1. Scope Construction
1.1 Future prompts shall state the governed clause IDs before file changes.
1.2 Future tasks shall remain scoped to the smallest file set required by the law.
1.3 Multi-domain work shall be split unless the prompt explicitly requires integration.

## 2. Completion Law
2.1 Completion shall require evidence.
2.2 Completion shall require control-point consultation where applicable.
2.3 Completion shall require audit triggering where a governed check exists.
2.4 Completion shall not rely on "it should work now" language.

## 3. Failure and Reopen Law
3.1 Failed controls shall not be skipped.
3.2 Prerequisite law shall be complete before phase jumps.
3.3 Reopen shall occur when retest fails or evidence is missing.

## 4. Clause IDs
4.1 Stable clauses in this document use the prefix `SYS-EXE`.

## 5. Enforceable Clauses
- `SYS-EXE-001`: Prompts shall name governing clause IDs before edits.
- `SYS-EXE-002`: Tasks shall remain scoped.
- `SYS-EXE-003`: Completion shall require evidence.
- `SYS-EXE-004`: Failed controls shall not be skipped.
- `SYS-EXE-005`: Reopen shall occur on failed or missing retest evidence.
