# Full-System Governance Integration

## 1. Purpose
1.1 This document shall bind all rewritten governance packs into one enforceable operating framework.
1.2 The platform shall be governed as a system of law, not a collection of disconnected documents.

## 2. Binding Hierarchy
2.1 The binding hierarchy shall be:
2.1.1 Constitution
2.1.2 Master Design
2.1.3 Standards and module constitutions
2.1.4 Control Point Engine
2.1.5 Audit Engine
2.1.6 RCA and Corrective Loop
2.1.7 Architect Dashboard
2.1.8 Execution Governance Rules

## 3. Supremacy and Conflict
3.1 The Constitution shall be supreme.
3.2 No competing governance source shall override the hierarchy.
3.3 No implementation prompt shall override governing law.
3.4 No runtime success claim shall override audit failure.
3.5 No audit result shall override the Constitution.

## 4. Clause IDs
4.1 Stable clauses in this document use the prefix `SYS-GOV`.

## 5. Enforceable Clauses
- `SYS-GOV-001`: The platform shall be governed by one source-of-truth hierarchy.
- `SYS-GOV-002`: Governance sources shall not compete.
- `SYS-GOV-003`: Implementation prompts shall remain subordinate to governing law.
- `SYS-GOV-004`: Runtime success claims shall not override audit failure.
- `SYS-GOV-005`: Audit results shall not override the Constitution.
