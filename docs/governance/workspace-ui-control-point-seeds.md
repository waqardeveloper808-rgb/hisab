# Workspace UI Control-Point Seeds

Updated: 2026-04-23
Status: Seed document for future workspace UI control-point generation.

## 1. Seed Scope

- `UX-SEED-001`: These seeds shall be used to generate future workspace UI control points.
- `UX-SEED-002`: These seeds shall remain subordinate to the workspace constitution and route ownership law.

## 2. Seed Themes

### 2.1 Route truth

- `UX-SEED-010`: Route truth must confirm every claimed route has a real owner.
- `UX-SEED-011`: No fake placeholder route may be accepted as shipped.

### 2.2 Page ownership

- `UX-SEED-020`: Real page ownership must map each sidebar item to an operational page.
- `UX-SEED-021`: Generic overview pages must not be used to hide missing modules.

### 2.3 Responsive continuity

- `UX-SEED-030`: Browser resize continuity must preserve hierarchy and primary actions.
- `UX-SEED-031`: Multi-panel collapse and reflow must remain predictable.

### 2.4 Panel behavior

- `UX-SEED-040`: Adjustable surfaces must support collapse, expand, close, restore, and resize where appropriate.
- `UX-SEED-041`: Panel state must not corrupt workflow state.

### 2.5 Record continuity

- `UX-SEED-050`: Register to preview detail continuity must preserve the active record and route context.
- `UX-SEED-051`: No dead-end preview state may be accepted.

### 2.6 Density and interaction

- `UX-SEED-060`: Information density must remain efficient and premium.
- `UX-SEED-061`: Interaction state consistency must cover hover, focus, active, loading, empty, error, success, and disabled states.

### 2.7 Future compatibility

- `UX-SEED-070`: The same shell law must extend to accounting, document, inventory, VAT, reporting, HR, timesheet, and payroll workspaces.
