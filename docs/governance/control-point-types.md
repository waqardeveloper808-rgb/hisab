# Control Point Types

## 1. Type Categories
1.1 Structural
1.2 Financial
1.3 Compliance
1.4 Relational
1.5 UI/UX
1.6 Workflow

## 2. Type Rules
2.1 Structural control points shall validate presence, shape, and governed structure.
2.2 Financial control points shall validate monetary and posting integrity.
2.3 Compliance control points shall validate jurisdictional and regulatory rules.
2.4 Relational control points shall validate source-to-target links and lineage.
2.5 UI/UX control points shall validate governed interaction state only when paired with real business truth.
2.6 Workflow control points shall validate stage transitions and blocked paths.

## 3. Clause IDs
3.1 Stable clauses in this document use the prefix `CPE-TYPE`.

## 4. Enforceable Clauses
- `CPE-TYPE-001`: Structural control points shall validate governed structure.
- `CPE-TYPE-002`: Financial control points shall validate monetary integrity.
- `CPE-TYPE-003`: Compliance control points shall validate regulatory rules.
- `CPE-TYPE-004`: Relational control points shall validate lineage.
- `CPE-TYPE-005`: UI/UX control points shall not replace business truth.
- `CPE-TYPE-006`: Workflow control points shall validate stage transitions.
